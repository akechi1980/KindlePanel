# 项目说明

这是一个为旧安卓平板设备开发的全屏看板应用项目，目标设备为 Kindle Fire HD 8 第7代一类老设备。请始终优先考虑稳定性、兼容性、轻量化和长期可维护性。

# 开发原则

1. 优先保证可运行和稳定，不要优先追求炫技。
2. 所有 UI 文案必须使用中文。
3. 所有功能都要考虑旧设备性能。
4. 避免不必要的复杂依赖。
5. 避免使用过新的 Android API。
6. 不依赖 Google Play Services。
7. 动画必须轻量。
8. 所有新增代码要尽量模块化。
9. 每次修改尽量保持变更范围清晰。
10. 不要擅自扩展需求。

# 功能边界

当前项目只聚焦以下三个主要模式：
- 本地看板模式
- 网页模式
- 相册模式

不要自行添加以下复杂能力，除非明确要求：
- 云同步
- 登录系统
- 远程后台
- 复杂账户体系
- 视频播放
- 重型数据库
- 复杂手势系统
- 复杂 3D 动画

# 界面要求

1. 应用默认全屏。
2. 主界面平时不显示按钮。
3. 用户点击空白区域三次后显示浮层菜单。
4. 浮层菜单只有“退出”和“设定”两个主要入口。
5. 所有界面必须适合深色背景展示。
6. 天气图标必须统一风格，不要杂乱。

# 模式要求

## 本地看板模式
左侧：
- 日期
- 星期
- 大号翻页时钟

右侧：
- 天气图标
- 当前温度
- 天气描述
- 最高温 / 最低温
- 城市名

## 网页模式
- 全屏 WebView
- 可配置链接
- 支持自动刷新
- 加载失败时提供中文提示

## 相册模式
- 显示本地图片
- 支持顺序 / 随机
- 支持切换间隔设置
- 使用轻量切换动画

# 常亮要求

应用前台运行时，应尽量保持屏幕常亮，减少自动休眠影响。
请把这部分逻辑集中封装，不要把常亮控制散落到各处。

# 配置要求

所有配置应可持久化保存，至少包括：
- 默认模式
- 网页地址
- 网页刷新间隔
- 天气城市
- 天气刷新间隔
- 相册目录
- 相册切换间隔
- 相册播放方式
- 是否保持常亮
